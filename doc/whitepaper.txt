OS64白皮书(v1.0) 

OS64是一个基于区块链技术的64位操作系统(区块链名称:OS64)。
OS64链是一个自洽的公链系统,发行一定数量的代币(Token),用于支持研发、测试和App下载等。
OVM是基于OS64链的全网单实例虚拟机,用于处理Token、NFT、DApp、挖矿等交易。
OS64是开源且免费使用的,其源代码(及初始编译工具)都将在区块链中保存。
OS64代币是OS64操作系统、OS64链和OVM发行的唯一代币。
OS64总发行量为2100亿枚,目前已在以太坊上发行,合约地址:代币名称:OS64。

系统角色:
核心团队:由创始人和初期支持者组成的DAO组织,负责投资、开发、产品、设计、测试、商业拓展等。
开发团队:由核心团队认可的个人、小团队或DAO组织,负责系统中某个模块或内置应用的研发和设计。
前瞻投资者:对OS64系统的每笔投资或捐赠以ETH为基准,其他形式的捐赠也将换算为ETH等值来衡量。所有的捐赠将和OS64代币一同用于支付核心团队和开发团队的报酬。
应用开发者:全球任意研发团队,可基于OS64操作系统规范研发开源或非开源的商业或免费应用,经DAO组织审核后可上架到应用商店。
DApp开发者:全球任意研发团队,可基于OVM的规范研发DApp应用,经DAO组织审核后可上架到DApp商店。
用户:OS64操作系统、链及代币的使用者.在OS64v1.0发布后，可以免费使用操作系统，并可利用闲置的存储及带宽资源参与挖矿。

路线图:
我们将OS64系统1.0版(含操作系统、链和挖矿等功能)上线前称为1.0研发期。
OS64 v1.0发布时,所有的代币数据将从ETH链映射并迁移到OS64链中。
在整个1.0研发期,OS64流通量将逐渐从0增加至最多10%(即210亿);
系统再保留10%的代币用于后续100年的开发及维护工作，其余80%将只能通过后续100年挖矿产出。
发放给开发团队及核心团队的代币，也有部分具有锁定期，以方便进行Bug管控。
因此,流通量与获得的投资或捐赠相关,也与研发进度相关。
我们会将1.0研发期分成多个阶段,具体划分将逐渐公布,请持续关注我们。
我们同时也欢迎使用大模型AI进行开发、设计、测试工作的团队来加入我们（内核的编程语言为Rust）。

投资和捐赠:
现在就是参与的最佳时机。每捐赠任意数量的ETH,我们将回赠一定数量的OS64代币作为回馈。
接受捐赠的ETH钱包地址：0x898b7Ebbd39217b5703cD9B5701dcE93B547b488
具体数量将根据后续公布的公式计算得出。
在整个1.0研发期,我们用于回赠的OS64代币不会超过100亿枚。
所有收到的捐赠或投资的ETH以及前210亿未用于回馈的代币将按照一定标准全部奖励到开发团队及核心团队。
更详细的奖励于开发及核心团队的措施将在后续持续补充。

公式：
在公式中，将用到如下常量： a = 9.9E-7 及 q = 1 + 3.3E-10
变量：当前已发行量(含锁定量)，记为：n = 当前捐赠及投资回馈总量 + 当前开发团队奖励总量
我们有以下公式(结果取到小数点后四位)，其中捐赠投资量的单位是ETH，回馈量是OS64代币数量：
    回馈量 = ln( 捐赠投资量 * (q-1) / a + q ^ (n + 1) ) / ln(q) - n
注：当 n = 0 时，代入 1 ETH，得到 1009932.6139 OS64，则说明计算正确。
当 n = 1009932.6139 时，代入 1 ETH，得到 1009596.1380 OS64，则说明计算正确。

注：根据该公式，顺利的话，从第一个币到v1.0上线这将很可能是一个千倍币。

OS64操作系统、区块链与挖矿:
操作系统将从底层支持NFT,如某个NFT图片,拥有者可以选择其他用户无法显示、添加水印显示、链接方式显示(点击后跳转到该NFT资产的版权介绍页)。
操作系统将从底层支持文件存储,如操作系统和内置应用的所有源代码将存放于区块链中,并向贡献出存储和带宽资源的用户提供挖矿奖励。
应用商店的数据也将存储于区块链中;甚至操作系统使用的任何公共数据也将存储于区块链中;并且许多公益数据也将存储于区块链中,如濒临灭绝动物的DNA样本,经授权的历史气象数据等。

这是一个理想型的操作系统和存储型区块链;希望能存储人类共同的理想,并成为大多数人愿意使用的操作系统。
